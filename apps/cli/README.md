# mixctl-cli

Command-line client for the mixctl daemon. Communicates over D-Bus using the proxy generated by `mixctl-core`.

## Usage

```bash
# Status
mixctl-cli ping
mixctl-cli status                           # PipeWire connection + default input

# Input management
mixctl-cli input list
mixctl-cli input get <id>
mixctl-cli input add <name> <color>
mixctl-cli input remove <id>
mixctl-cli input set-name <id> <name>
mixctl-cli input set-color <id> <color>
mixctl-cli input move <id> <position>
mixctl-cli input get-default
mixctl-cli input set-default <id>           # 0 to clear

# Output management
mixctl-cli output list
mixctl-cli output get <id>
mixctl-cli output add <name> <color> <source_output_id>  # 0 for defaults
mixctl-cli output remove <id>
mixctl-cli output set-name <id> <name>
mixctl-cli output set-color <id> <color>
mixctl-cli output set-volume <id> <0-100>
mixctl-cli output set-mute <id> <true|false>
mixctl-cli output set-target <id> <device_name>  # link to physical device
mixctl-cli output move <id> <position>

# Route management (input→output matrix cells)
mixctl-cli route list <output_id>
mixctl-cli route get <input_id> <output_id>
mixctl-cli route set-volume <input_id> <output_id> <0-100>
mixctl-cli route set-mute <input_id> <output_id> <true|false>

# Audio stream assignment
mixctl-cli stream list                      # active PipeWire streams
mixctl-cli stream assign <pw_node_id> <input_id> [--remember]

# App rules (auto-assign streams to inputs)
mixctl-cli rule list
mixctl-cli rule set <app_name> <input_id>   # exact or glob pattern
mixctl-cli rule remove <app_name>

# Hardware capture devices
mixctl-cli capture list                     # available mics/line-in
mixctl-cli capture add <pw_node_id> <name> <color>

# Config sections (get/set component configuration)
mixctl-cli config get beacn              # JSON output
mixctl-cli config get ui
mixctl-cli config set beacn '{"layout":"dial","dial_sensitivity":3}'
mixctl-cli config set applet '{"poll_interval_ms":50}'

# Pagination (for hardware display)
mixctl-cli page get
mixctl-cli page set <page>

# Live signal monitoring (runs until Ctrl-C)
mixctl-cli listen all       # all signals
mixctl-cli listen state     # output volume/mute + route changes
mixctl-cli listen config    # input/output additions/removals
mixctl-cli listen page      # page changes
```

### Notes

- Inputs and outputs share a single ID space — IDs are unique across both.
- `output add` takes a `source_output_id` to copy route volumes from an existing output. Use `0` for default routes (all at 100%, unmuted).
- `output set-target` links an output's audio to a physical device (e.g., headphones). Pass an empty string to clear.
- `stream assign --remember` saves the assignment as an app rule so future streams from that app are auto-assigned.
- App rule patterns support glob syntax (e.g., `firefox*`, `*spotify*`).
- Config sections (`beacn`, `ui`, `applet`, `cli`) are exchanged as JSON. Use `config get` to see current values and `config set` to update. Changes persist to `~/.config/mixctl.toml` and emit a `config_section_changed` signal.

## Dependencies

- `mixctl-core` — `MixCtlProxy`, shared types, config section types
- `serde` / `serde_json` — config section serialization
- `zbus` — D-Bus session connection
- `tokio` — async runtime
- `clap` — argument parsing
- `futures-lite` — signal stream combinators (for `listen` command)
