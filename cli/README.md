# mixctl-cli

Command-line client for the mixctl daemon. Communicates over D-Bus using the proxy generated by `mixctl-core`.

## Usage

```bash
# Check daemon is running
mixctl-cli ping

# Channel queries
mixctl-cli channel list
mixctl-cli channel get <id>

# Channel management
mixctl-cli channel add <name> <color>
mixctl-cli channel remove <id>
mixctl-cli channel set-name <id> <name>
mixctl-cli channel set-color <id> <color>
mixctl-cli channel move <id> <position>

# Volume and mute
mixctl-cli channel set-volume <id> <volume>
mixctl-cli channel set-mute <id> <true|false>

# Pagination
mixctl-cli page get
mixctl-cli page set <page>
```

Channels are identified by their integer ID. Use `channel list` to see IDs. The `add` command auto-assigns the next unused ID and prints it.

## Dependencies

- `mixctl-core` — `MixCtlProxy` and `ChannelInfo`
- `zbus` — D-Bus session connection
- `tokio` — async runtime
- `clap` — argument parsing
